# Notion Flow

[English](README.md) · [中文示例文档](examples/notion-flow-demo.zh.md) · [English example note](examples/notion-flow-demo.md)

为 Obsidian 提供 Notion 风格的块编辑体验：移动完整的 Markdown 内容块、用 `/` 插入内容、通过浮动工具栏设置文本格式，并在保留纯文本工作流的同时增强表格编辑。

> [!NOTE]
> Notion Flow 只增强 Obsidian 内的编辑体验，不连接 Notion，也不提供 Notion 的导入、导出或同步功能。

## 功能概览

- **内容块操作：**拖动段落、标题、带子项的列表、引用、Callout、代码块和表格；点击手柄打开块菜单，点击 `+` 在下方插入内容。
- **多块框选：**从编辑区空白处拖出选框（或按住 Alt/Option 从任意位置拖动），一次完成转换、复制、剪切、粘贴替换、创建副本或删除。
- **斜杠命令：**用中英文关键词插入标题、列表、Callout、折叠块、代码块、表格、分栏、分割线、嵌入和内部链接。
- **分栏：**像 Notion 一样让块并排显示——用「/分栏」插入、在块菜单中转换，或把块拖到另一个块的右缘。以普通嵌套 Callout 语法写入，笔记保持可移植。
- **批注：**像 Notion 一样选中文本添加批注——黄色锚点、💬 标记，点击即可查看、编辑或解除。批注保存在笔记内部，在其他 Markdown 应用中不可见。
- **格式工具栏：**设置粗体、斜体、下划线、删除线、文字颜色、高亮、行内代码和链接，也可以清除格式——在代码块内同样可以加粗、斜体、划线和变色。
- **表格工具：**增删、对齐、着色、格式化和移动表格；编辑原始 Markdown 表格时可使用快捷导航。
- **更简洁的实时预览：**优化任务、引用、分割线、行内代码、Mermaid 图表、嵌套块和 Markdown 标记的显示；列表符号循环独立保持启用。

## 使用要求

- Obsidian 1.5.0 或更高版本
- 仅支持 Obsidian 桌面端，不支持移动端
- 完整编辑体验需要实时预览模式
- 不需要账号、API Key 或外部服务

## 安装

### 从已发布的 Release 安装

当 [Releases 页面](https://github.com/xinli12/obsidian-notion-flow/releases)存在已发布版本时使用此方式；如果页面尚无可用版本，请按下方步骤从源码构建。

1. 从同一个 Release 下载 `main.js`、`manifest.json` 和 `styles.css`。
2. 新建 `<库>/.obsidian/plugins/notion-flow/` 文件夹。
3. 将三个文件复制到该文件夹。
4. 重启 Obsidian。
5. 打开**设置 → 第三方插件**，按需启用第三方插件，并开启 **Notion Flow**。

### 从源码构建

需要 Node.js 18 或更高版本；CI 使用 Node.js 20。

```bash
git clone https://github.com/xinli12/obsidian-notion-flow.git
cd obsidian-notion-flow
npm install --legacy-peer-deps
npm run build
```

将生成的 `main.js` 与 `manifest.json`、`styles.css` 一起复制到 `<库>/.obsidian/plugins/notion-flow/`，重启 Obsidian，然后在**设置 → 第三方插件**中启用插件。

在**设置 → 编辑器 → 默认编辑模式**中选择**实时预览**，也可以从当前笔记的视图菜单临时切换。

## 快速上手

1. 将[中文](examples/notion-flow-demo.zh.md)或[英文](examples/notion-flow-demo.md)示例文档复制到你的库中。
2. 使用实时预览打开副本。
3. 悬停到内容块左侧：拖动 `⋮⋮` 移动，点击 `⋮⋮` 打开菜单，点击 `+` 在下方插入新块。
4. 在行首输入 `/`，选择要插入的块类型。
5. 选中文本，打开格式工具栏。
6. 点击已渲染表格中的单元格，打开表格工具栏。

大多数编辑和外观功能都可以在**设置 → Notion Flow** 中调整。

## 使用 Notion Flow

### 内容块

拖拽手柄会按完整的 Markdown 结构处理内容。列表项会带着嵌套子项一起移动；引用、Callout、代码块和表格会作为整体移动。拖动时横向移动指针可选择合法的嵌套层级；只横向拖动可以原地调整缩进。拖到编辑区上、下边缘会自动滚动，按 `Esc` 取消。

单击手柄而不拖动，可打开块菜单：

- 将单行块转换为正文、标题、列表、待办或引用。
- 创建副本、复制文本或删除完整内容块。
- 对引用和 Callout 选择标注类型、切换可折叠状态,或将 Callout 还原为普通引用。
- 对表格在顶部或底部加行、在左侧或右侧加列，并设置整表对齐、表格底色或格式化源码。

点击 `+` 会在当前块下方插入空行并打开斜杠菜单。

从行旁空白处拖动可框选多个块；也可以按住 Alt/Option，从编辑区任意位置开始拖动。浮动块工具栏会一次转换所有选中的普通块（表格、围栏代码块和多行引用会被跳过，以免损坏其结构），并提供复制与删除按钮。框选后也支持键盘操作：`Cmd/Ctrl+C` 复制，`Cmd/Ctrl+X` 剪切，`Cmd/Ctrl+V` 用剪贴板内容替换选中块，`Cmd/Ctrl+D` 创建副本并选中，`Cmd/Ctrl+A` 扩展为全选所有块，`Backspace`/`Delete` 删除。按 `Esc` 或点击 × 可清除选择。

### 斜杠命令

在行首、空白字符后，或直接在中日韩文字后输入 `/`（全角 `／` 同样触发）。无论界面语言是什么，都可以使用中英文关键词搜索。每个菜单条目显示图标、名称、一行说明和对应语法；未输入关键词时，最近使用过的命令会排在最前。

可用项目包括 `/h1`、`/h2`、`/h3`、`/bullet`、`/number`、`/todo`、`/quote`、`/callout`、`/toggle`、`/cols2`、`/cols3`、`/code`、`/table`、`/divider`、`/image` 和 `/link`。

用鼠标选择**表格**后会打开最大 10 × 10 的尺寸网格。可以在网格中拖动，也可以用方向键调整后按 `Enter`。如果直接在斜杠结果上按 `Enter`，则插入默认的 3 × 3 表格。代码围栏内不会触发斜杠菜单。

### 文本格式

选中编辑器文本即可显示浮动工具栏，支持：

- 粗体、斜体、下划线、删除线和行内代码
- 文字颜色和高亮色板
- Markdown 链接
- 对选区添加批注（💬）
- 清除格式（`Cmd/Ctrl+\`）：清掉与选区相关的所有格式——粗体/斜体（`*` 与 `_` 均可）、删除线、高亮、行内代码，以及隐藏 HTML 标签完全在选区之外的颜色和下划线。标记永远整对移除，部分选中也不会留下孤立的 `**` 或 `</span>`。批注是内容而非格式，会完整保留。

启用**粘贴 URL 生成链接**后，在选中的单行文本上粘贴 `http://`、`https://` 或 `obsidian://` URL，会自动生成 `[文本](链接)`。启用**粘贴 URL 自动获取标题**后，未选中文本直接粘贴 URL 会先立即插入原始链接，随后在后台获取网页标题并升级为 `[网页标题](链接)`；无法访问网页时保留原始 URL，粘贴到代码环境时不做任何处理。

在代码块内，工具栏会自动改用 HTML 标签（`<b>`、`<i>`、`<s>`、`<u>` 与颜色 span），因为 Markdown 标记在围栏代码中只是普通文本。这些标签在实时预览中隐藏、在阅读视图中渲染为带样式的代码；行内代码和链接按钮在代码块内会置灰（在代码中没有意义）。在代码块内**清除格式**只移除这些标签——代码里真实的 `*`、`` ` `` 等字符绝不会被误删。

### 代码块

启用**代码块编辑增强**后：

- 按 `Enter` 续行时保持当前行缩进——嵌套在（深层）列表中的代码块，每个新行都自动对齐到列表的内容列。
- 在代码文本开头按 `Backspace` 一次回退一级缩进，而不是逐字符删除。
- 在刚输入的、未闭合的 ` ``` ` 行末按 `Enter` 会自动补全闭合围栏并把光标放进块内，代码块不会再"吞掉"下方的整篇笔记。
- 按 `Cmd/Ctrl+Shift+Enter`（**跳出代码块**命令，可在快捷键设置中重新绑定）跳出代码块，落到其下方一个缩进正确的新行（若缺少闭合围栏会先补上）。

### 标注与引用

启用**标注与引用编辑增强**后：

- 在引用或 Callout 内按 `Enter` 会自动续写 `>` 标记；在只有 `>` 的空行按 `Enter` 会退出块（嵌套时逐层退出），在 Callout 末尾连按两次即可离开，与 Notion 一致。
- 在行首正文位置按 `Backspace` 会一次移除一层 `>` 标记，而不是逐字符删除。
- 在引用或 Callout 内粘贴多行文本时，每一行都会自动补全前缀，块不会被撕开；富文本会先转换为 Markdown，文件粘贴保持 Obsidian 原有行为。
- 点击已渲染的 Callout 会把光标直接放到点击处开始编辑，替代 Obsidian 原生"第一次点击选中整块"的行为；拖拽仍可选中整块。
- 光标位于 Callout 内时，展开的源码行保留渲染外观：按类型着色的背景、圆角和彩色标题行。
- 在实时预览中点击已渲染 Callout 的图标，会打开类型菜单，包含 Obsidian 内置的十三种类型、**可折叠**开关和**转换为引用**。块手柄菜单提供相同操作；对普通引用选择类型会直接升级为 Callout。
- 开启**更简洁的所见即所得渲染**后，未编辑的代码块围栏行会隐藏 ``` 标记（语言标签仍然显示），引用标记在活动行也保持安静。

### Mermaid 图表

启用**更简洁的所见即所得渲染**后，实时预览和阅读视图中的 Mermaid 图表会使用适配当前主题的卡片式画布。流程图、时序图、子图、注释、甘特图和饼图拥有更清晰的视觉层级。宽图会保留可读字号并支持横向滚动，不再被压缩成难以辨认的缩略图；聚焦图表后可使用键盘或触控板滚动。Mermaid 源码不会被修改。

### 分栏

启用**分栏**后，块可以像 Notion 一样并排放置：

- 输入「/分栏」（或 `/cols2`、`/cols3`），选择**两栏**或**三栏**。
- 用手柄把一个块拖到另一个顶层块的**右缘**——出现竖向的强调色指示条即为目标——放下后两个块并排显示。拖放到已有分栏行上会追加一栏。
- 每个已渲染的分栏行右上角都有一个小分栏按钮（悬停时更明显），点一下即可**添加一栏**、调**栏宽**或**拆开分栏**；块手柄菜单提供相同操作，并对任意顶层块提供**转为分栏**。
- 直接拖动两栏之间的分隔线即可调宽，拖动时会显示实时百分比；整次拖动只产生一条撤销记录。用 Tab 聚焦分隔线后，←/→ 每次调整 1%（按住 Shift 为 5%），双击恢复均分。
- 悬停单独一栏会出现该栏的 **⋯** 菜单，可在左右添加新栏、左右移动当前栏或删除。删除非空栏前会确认；两栏删去一栏后，保留内容会自动拆回普通块。
- 用标注元数据固定栏宽：`> > [!nf-col|30]` 占据整行的 30%（有效值 10–90），未指定宽度的栏平分其余空间；也可以在块菜单的**栏宽**子菜单里选预设（均分／左窄／右窄）。布局按 Obsidian 窗格宽度响应，低于 560px 时整组纵向堆叠，三栏不会出现不稳定的“2+1”换行。
- 块菜单里的**拆开分栏**可以把分栏行还原为依次排列的普通块。
- 空栏会显示虚线 **+** 占位，可见且可点击；悬停已渲染的分栏行会勾勒出每一栏的边界。
- 点击已渲染的一栏即可就地编辑，不再暴露整行的 `> >` 结构前缀。当前栏进入干净编辑态，其他栏仍以阅读态显示作为上下文；点击另一栏预览即可切换，用源码按钮查看原始语法，按 `Esc` 或勾选按钮完成。撤销／重做仍统一属于当前笔记。
- 在栏内（或任何引用/Callout 内）使用斜杠命令时会保留 `>` 标记，插入的代码块、表格、Callout 都留在原栏中。
- 分栏以嵌套 Callout 语法写入——`[!nf-cols]` 包裹若干 `[!nf-col]` 子块——在其他 Markdown 应用中显示为普通嵌套引用，笔记不会被插件绑定。
- 需要时仍可在源码模式查看可移植的 `[!nf-cols]` / `[!nf-col]` 语法；结构行以淡色显示，因为它们是结构而非正文。
- 分栏源码现在按真实的“行／栏”结构解析：代码围栏中的 `[!nf-col]` 不会被误认成新栏，混合 Tab 与空格的缩进不会在转换时变形，空栏中的 Enter／Backspace 不会删坏结构标记，栏内代码块和表格的键盘编辑也会保留栏前缀。

### 批注

启用**批注**后，可以像 Notion 一样选中文本加注释：

- 通过浮动工具栏的 💬 按钮、**添加批注**命令或 `Cmd/Ctrl+Shift+M` 添加批注。批注针对单行内的选区。
- 锚文本以黄色高亮，其后跟一个小 💬 标记。悬停可查看批注内容；点击标记可编辑，或选择**解除批注**（解除后标记消失、正文保留）。
- 阅读视图中锚点保持高亮，悬停显示批注。
- 批注以 `<span class="nf-cmt" data-nf-cmt="…">正文</span>` 的形式保存在笔记内——在其他 Markdown 应用中锚文本正常显示、批注内容不可见。**清除格式**也会一并移除选区内的批注标记。

### 表格

在实时预览中点击已渲染的单元格，可使用行列操作、当前列对齐、单元格或表格底色、格式化和删除。按 `Alt+F10` 聚焦工具栏，再用左右方向键在按钮之间移动。

**表格编辑增强**设置只作用于源码模式中的原始 `|` 表格，或仍未渲染的表格：

- `Tab` / `Shift+Tab` 在可编辑单元格之间移动。
- `Enter` 移动到下一行的同一列。
- 从最后一个单元格继续导航会自动追加一行。
- 输入 `| 名称 | 数量` 这样的表头后按 `Tab`，会自动创建分隔行和第一行表体。
- 在空的最后一行按 `Enter` 会移除该行并离开表格。

命令面板和原始表格的右键菜单也提供相同的结构操作。**格式化表格**会按中日韩字符的显示宽度对齐各列。

### 外观

- **更简洁的所见即所得渲染**是一组纯显示样式：优化标题、任务、分割线、行内代码、引用和 Mermaid 图表，但不会改写 Markdown；列表循环独立保持启用。实时预览中，非活动行的引用标记会隐藏，进入活动行后恢复显示。
- 实时预览和阅读视图都会按完整列表层级循环样式：无序列表为实心圆、空心圆、方块；有序列表为数字、小写字母、小写罗马数字。实时预览的活动行仍显示可编辑的 Markdown 编号。
- 拖入列表任意深度的代码块、Callout 和引用会使用真实内容层级，并在源码态与渲染态保持对齐。
- **Notion 风格表格**、表头底色和斑马纹可以分别设置。
- 阅读视图中的宽 Markdown 表格可以横向滚动。
- **隐藏行内 Markdown 标记**可在光标未进入源码时隐藏格式标记；Markdown 链接始终可见，源码模式不受影响。

| 操作 | 默认快捷键 |
| --- | --- |
| 上移内容块 | `Alt/Option+↑` |
| 下移内容块 | `Alt/Option+↓` |

| 引用条颜色 | 文字颜色 | 中性文字色、强调色、主题默认或九种调色板颜色 |
| 行内代码颜色 | 红 | Notion 风格的 `行内代码` 文字颜色、主题默认或九种调色板颜色 |
| 代码块主题 | 主题默认 | 十种内置选项（包括 Obsidian 自适应）；支持的配色会自动切换明暗版本 |
| 隐藏行内 Markdown 标记 | 开 | 在实时预览中隐藏支持的行内格式标记 |

设置页还提供**恢复默认设置**。

## 会写入 Markdown 的内容

大多数操作会生成标准 Markdown。下划线、文字颜色和彩色高亮使用内联 HTML，代码块内的所有格式（`<b>`、`<i>`、`<s>`、`<u>`、颜色 span）同样如此；分栏是嵌套 Callout（`[!nf-cols]` / `[!nf-col]`），在其他应用中降级为普通嵌套引用；批注是带 data 属性的 span，在其他应用中锚文本正常显示、批注内容不可见；表格和单元格颜色使用由 Notion Flow 样式表渲染的少量 class 标记。启用**粘贴 URL 自动获取标题**时，粘贴链接会向该 URL 发送一次请求以读取标题；关闭该设置即可完全离线。这些标记会保留在笔记中，因此可以随同步和复制粘贴保存；但在 Obsidian 之外或未启用插件时，不保证保留相同的颜色外观。

Notion Flow 不会上传笔记内容。打开文档或示例时会使用浏览器，**复制文本**功能会使用系统剪贴板。

## 故障排查与限制

- **没有块手柄：**请使用桌面端实时预览，并确认已启用**块拖拽**。
- **没有表格工具栏：**请启用**浮动格式工具栏**。**表格编辑增强**控制的是原始表格快捷键和右键菜单，不控制该工具栏。
- **`Tab` 或 `Enter` 没有使用 Notion Flow 导航：**增强快捷键只作用于原始 Markdown 表格，不作用于 Obsidian 已渲染的单元格编辑器。
- **工具栏无法格式化渲染组件内的选区：**Obsidian 在已渲染 Callout 或彩色文字中可能只提供 DOM 选区。请先打开组件源码，或把选区扩展到普通编辑器文本。
- **旧笔记中的嵌套 Callout 显示成代码：**将光标放在 Callout 源码上，执行**Notion Flow: 修复列表内 Callout**。该修改可以撤销。
- **手动安装后找不到插件：**确认目录为 `.obsidian/plugins/notion-flow/`，其中包含三个发布文件，然后重启 Obsidian。

## 语言

界面会跟随 Obsidian 的语言设置，内置英文和简体中文。无论使用哪种界面语言，斜杠搜索都接受中英文关键词。**设置 → Notion Flow → 帮助与示例**中提供两种语言的文档和示例按钮。
